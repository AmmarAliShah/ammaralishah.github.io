What it took to start this blog

Written by Ammar Shah

In the previous post I mentioned that I had to face some problems to set up this blog and as promised, I’m going to be explaining those now.

WordPress my first choice

As soon as I thought of starting this blog, the CMS that came into my mind was WordPress. After all, I had been using WordPress for over an year in other blogs and was comfortable with its interface. I set up WordPress easily on my server and got started. I wanted to find a good looking theme so I started searching the web and found a few that I liked. At this point my troubles started. I couldn’t install any of the themes or plugins, WordPress gave me an error each time. Another thing I noticed that WordPress was also slow. This was enough of a reason to make me switch.

Ghost

My second solution for CMS was Ghost. I had never used Ghost before but had seen it in action and really liked the clean, content focused interfaced. I excitedly went over to the Ghost site downloaded the code and uploaded it to my server. When i went to set up Ghost, I found out that instead of a setup page, I was getting a file directory. I hurriedly went over to my old pal Google and just in a few searches learned that Ghost is a JavaScript based CMS, unlike WordPress which is in PHP, and does not work on shared-hosting services such as the one I was/am using. I was heartbroken. I tried many workarounds to it but failed in them all. So I just gave up on Ghost and came back to WordPress only to face the same problems as before.

MODX

After my failure with Ghost, I did what anyone would do and googled for other popular CMS. I found many and also gave them a try but I didn’t like any one of them: some had a bad UI, some had a bad UX, some had bad both and some left me to write most of the code. I got fed up with this and just decided to either use WordPress or postpone this blog for later. Tired, I fired up Twitter and began scanning through tweets re-tweeting where needed. I was doing this when I stumbled upon a link to MODX. I was desperate so I downloaded it and gave it a try. It was much more larger then WordPress and took much longer to upload. When it finally uploaded, I had problem setting it up – it would throw an error every time i tried to access it. I turned toward my good friend Google again and began reading. After just a few articles, I found my problem – I had to set up something I can’t remember. I went through the setup and was ready to blog. I liked the dashboard of MODX and was excited to start blogging but now here came the problem – where do i write? I went over to MODX documentation this time and began going through it. After five minutes, I realized that MODX was one of those CMS that wanted you to write your own code. Frustrated, I decided to postpone this blog.

Back to WordPress

I never really forgot about the blog: everyday i would google looking for new CMS but this time didn’t try them but Googled to see what other people though of them. I went with none of them though. After just a few days, I decided I can go with WordPress if I just use the default theme. I set up WordPress again but didn’t post anything and left it there.

Current state

This was about 2 or 3 months ago. My server still has Ghost and MODX uploaded in it, but I don’t care. I’m going to use WordPress till I move onto either Ghost or a static site generator. I could also build my own CMS but that can only be done in summer when I don’t have school to worry about. Right now, I’m running WordPress with its default theme. It isn’t bad but it isn’t great either. I’m going to try to switch as soon as I can so I don’t have to go over the process of transferring too much data to whatever CMS I’m going to use.

Well this is it – what it took to start this blog. I hope I do make good use of it after all the trouble of setting it up. I’m going to be making the most of this and styling/customizing this to my needs before I move onto another CMS.
